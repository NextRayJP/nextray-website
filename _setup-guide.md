## 1. Google Search Console の登録（所要15分・無料）

### 現状

所有権確認ファイル `google1866306a5a1aca43.html` は**すでにサイトに設置済み**で、
`https://nextray.jp/google1866306a5a1aca43.html` にアクセスすると正しく応答します。
つまり、過去に一度登録に着手された形跡があります。

以下の手順で、登録済みかどうかの確認と、未完了の場合の完了までを行えます。

### 手順

1. https://search.google.com/search-console/ にアクセスし、Googleアカウントでログイン
2. 左上のプロパティ選択メニューを開き、`nextray.jp` が既にあるか確認
   - **ある場合** → 手順5（サイトマップ送信）へ進む
   - **ない場合** → 「プロパティを追加」をクリックして手順3へ
3. プロパティタイプは **「URLプレフィックス」** を選び、`https://nextray.jp/` を入力
   - ※「ドメイン」タイプはDNS設定が必要なので、こちらの方が簡単です
4. 所有権の確認方法で **「HTMLファイル」** を選択
   - すでにファイルは設置済みなので、ダウンロードは不要
   - そのまま「確認」ボタンを押せば完了します
   - ※もし別のファイル名を要求された場合は、そのファイルをダウンロードして
     `nextray-website` フォルダ直下に置き、再度公開（git push）してから確認してください
5. **サイトマップを送信**（ここが本題）
   - 左メニューの「サイトマップ」を開く
   - 「新しいサイトマップの追加」に `sitemap.xml` と入力して送信
   - ステータスが「成功しました」になればOK（反映に数日かかることがあります）
6. 左メニューの「URL検査」でトップページのURLを入れ、「インデックス登録をリクエスト」
   - 主要ページ（トップ・料金・各サービス）は個別にリクエストしておくと早まります

### 登録後に見るべき指標

- **検索パフォーマンス** → どのキーワードで表示・クリックされているか
- **インデックス作成 > ページ** → 25ページが登録されているか、除外されていないか
- **エクスペリエンス > ウェブに関する主な指標** → 表示速度の問題

登録直後はデータが空です。1〜2週間ほどで蓄積されます。

## 4. コラムの継続について

`blog.html` に5本の記事を公開しました。
SEO上、この施策は**継続してこそ効果が出ます**。逆に更新が止まると
「放置されているサイト」という印象を与えます。

### 現実的な運用ペース

- **月1本**でも十分です。無理なら2ヶ月に1本でも構いません
- ネタは「ご相談の場でよく聞かれること」から拾うのが最も楽で、かつ検索需要とも合致します
- 既存記事の**加筆・更新**も有効です（`dateModified` を更新してください）

### 記事を追加する手順

1. `blog-meo-basics.html` をコピーして新しいファイル名にする
2. `<title>` `description` `canonical` `og:*` を書き換える
3. JSON-LD の `headline` `datePublished` `dateModified` `mainEntityOfPage` `BreadcrumbList` を書き換える
4. 本文（`.article-head` と `<article class="article">` の中身）を書く
5. `blog.html` の記事一覧に `.post-card` を1つ追加し、JSON-LD の `blogPost` 配列にも追加
6. `sitemap.xml` に URL を追加
7. git push

### 書くときの原則（重要）

- **統計や数値を創作しない**。「〜%の企業が」「調査によると」は出典がなければ書かない
- 実績・お客様の声・導入事例は、事実がある場合のみ
- 検索順位の保証はしない
- 売り込みより「読んだ人が自分で実行できること」を優先する
  （結果的にそれが信頼につながり、問い合わせにもつながります）

## 5. サイト全体のメンテナンス上の注意

- **共通部分（ヘッダー・フッター・ナビ）は全27ページに直接埋め込まれています。**
  ナビ項目を1つ変えるだけで全ページの修正が必要です。
  頻度が増えるようなら、共通部分を1ファイルにまとめる仕組みへの移行を検討してください。
- **CSSは2箇所にあります。**`index.html` はインライン `<style>`、
  その他のページは `assets/site.css` です。全体に効かせる変更は両方に入れてください。
- 料金を変更した場合、以下すべての更新が必要です：
  `index.html`（サービスカード）／`pricing.html`／各サービスページ／
  `faq.html`／各構造化データの `Offer`
